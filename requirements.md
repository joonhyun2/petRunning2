# Requirements

러닝 캐릭터 성장 앱 (petRunning2)

디자인 스펙 참조: `run_design/spec/`

---

## 앱 개요

달리기를 하면 캐릭터(펫)가 성장하는 모바일 앱.
달릴 때마다 경험치와 크레딧을 획득하고, 크레딧으로 펫 아이템을 꾸밀 수 있다.

---

## 화면 구성

| 화면 | 진입 | 역할 |
| --- | --- | --- |
| 홈 | 앱 시작, 하단 네비 | 펫 성장 상태 + 오늘 러닝 요약 + 달리기 시작 |
| 카운트다운 | 홈 `달리기 시작` (권한 허용 후) | 3초 카운트다운 후 Running 진입 |
| Running | 카운트다운 완료 | 달리기 중 실시간 측정 + 보상 피드백 |
| 결과(Result) | Running `종료` | 러닝 결과 요약 + 획득 보상 |
| 통계 | 하단 네비 | 주간/월간 통계 차트 + 보상 요약 |
| 꾸미기 | 하단 네비 | 펫 꾸미기 아이템 선택 + 적용 |
| 프로필 | 하단 네비 | 사용자 정보 + 배지 + 메뉴 |

---

## 기능 요건

### 홈

- 펫 이미지, 현재 레벨, 레벨 진행률(%)을 표시한다.
- 오늘의 러닝 요약(뛴 시간, 뛴 거리, 평균 페이스)을 표시한다.
  - 오늘 러닝 데이터가 없으면 수치를 `0` 또는 초기값으로 표시한다.
- `달리기 시작` 버튼으로 위치 권한 요청 → 카운트다운 화면 진입.

### 카운트다운

- 3, 2, 1 숫자 애니메이션 후 Running 화면으로 자동 전환된다.
- 하단 네비게이션을 표시하지 않는다.

### Running

- GPS로 현재 위치를 수신하여 페이스, 거리, 시간을 실시간으로 측정한다.
- 화면에 현재 페이스(크게), 거리, 시간을 표시한다.
- 달리는 동안 경험치와 크레딧 보상을 실시간으로 피드백한다.
- GPS 연결 상태를 헤더에 표시한다.
- 일시정지: 측정을 멈추고 재개 가능하다.
- 종료: 측정을 종료하고 결과(Result) 화면으로 이동한다.
- 달리는 중에는 하단 네비게이션을 표시하지 않는다.
- GPS 권한이 없거나 수신에 실패하면 에러 상태를 표시한다.

### 결과(Result)

- 종료된 러닝의 총 거리, 뛴 시간, 평균 페이스를 표시한다.
  - ※ 칼로리, 경로 지도는 현재 미구현.
- 이번 달리기에서 획득한 경험치와 크레딧을 표시한다.
- `완료` 버튼: 러닝 결과를 저장하고 홈으로 이동한다.
- `공유` 버튼: 결과를 외부 앱으로 공유한다.
- 결과 저장 중에는 완료 버튼을 비활성 처리한다.
- 결과 화면에서는 하단 네비게이션을 표시하지 않는다.

### 통계

- 주간/월간 기간을 토글로 전환한다.
- 선택 기간의 총 거리, 달린 횟수, 평균 페이스 요약을 표시한다.
- 요일별 거리 막대 차트를 표시한다.
- 요일별 페이스 추이 선 차트를 표시한다.
- 최고 페이스, 획득 크레딧, 획득 경험치 요약 카드를 표시한다.
- 기간 내 데이터가 없으면 수치를 `0`, 차트를 빈 값으로 표시한다.
- 기간 변경 시 로딩 상태를 처리한다.

### 꾸미기

- 펫 미리보기를 상단에 표시한다.
- 아이템 목록을 표시하고 선택하면 미리보기에 반영된다.
- 잠금 아이템은 가격 표시와 함께 구매 플로우를 안내한다.
- `적용` 버튼: 현재 선택한 꾸미기 설정을 저장한다.

### 프로필

- 사용자(펫) 이름, 레벨, 진행률을 표시한다.
- 획득 배지 목록을 표시한다. 배지가 없으면 빈 상태를 표시한다.
- 메뉴: 내 아이템, 알림 설정, 문의하기, 로그아웃.
- 프로필/배지 데이터를 불러오지 못하면 에러 상태를 표시한다.

## 네비게이션

```
홈 ──(달리기 시작 → 권한 허용)──▶ 카운트다운 ──▶ Running ──(종료)──▶ 결과(Result) ──(완료)──▶ 홈
                                                                              └─(공유)──▶ 공유 플로우

하단 네비: 홈 / 꾸미기 / 통계 / 프로필
  ※ 카운트다운, Running, 결과 화면에서는 하단 네비 미표시
```

---

## 시스템 요건

### 위치 / GPS

- 달리기 시작 전 위치 권한(Fine Location + Coarse Location)을 요청한다.
- 권한 거부 시 에러 안내를 표시한다.
- GPS 수신은 `LocationDataSource` (FusedLocationProviderClient + callbackFlow)로 처리한다.
- 현재 GPS 업데이트 주기: 3초 (최소 1.5초 간격).
- ※ Foreground Service 및 백그라운드 위치 유지는 미구현 (추후 과제).
- ※ Wake Lock (화면 꺼짐 대비)은 미구현 (추후 과제).

### 데이터 저장 (Room DB — `petrunning.db`)

- 로컬 Room DB를 사용하며 DataStore/SharedPreferences는 사용하지 않는다.
- **DogEntity** (table: `dog`): 펫 이름, 레벨, 현재 경험치, 최대 경험치, 크레딧.
- **RunRecordEntity** (table: `run_record`): 거리(km), 경과 시간(초), 페이스(초/km), 획득 경험치, 타임스탬프.
  - ※ 칼로리, 경로 좌표는 현재 미저장 (추후 과제).
- **OwnedItemEntity** (table: `owned_items`): 소유 아이템 ID, 장착 여부.
  - Badge 별도 테이블 없음 — 아이템 소유로 통합 관리.
- 앱이 종료/재시작되어도 데이터가 유지되어야 한다.

### 보상 계산

- 러닝 거리 기준으로 경험치와 크레딧을 계산한다.
- 경험치가 최대치(maxXp) 이상이면 레벨이 오른다.

### 공유

- 러닝 결과(거리, 시간, 페이스)를 Android Share Intent로 공유한다.

## 비기능 요건

- 기준 화면 폭 `394px`, 기기 폭에 따른 반응형 레이아웃.
- 최소 지원 Android: **API 24 (Android 7.0)** 이상.
- GPS 수신 주기: 현재 3초 (목표 1초 이하 — 추후 조정 예정).
- 폰트: Noto Sans KR.
- 색상은 `run_design/spec/design_system/colors.md`의 색상 토큰을 따른다.
- 다크 모드 미지원 (라이트 모드 고정).
